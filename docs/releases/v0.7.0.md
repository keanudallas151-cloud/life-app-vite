# v0.7.0 — Firebase platform migration and release hardening

## Summary

This release moves the active app runtime fully onto Firebase for authentication, profile sync, avatar uploads, and the Inventors & Investors networking flow. It also tightens release hygiene with a fixed ESLint 9 setup, safer local env handling, and a cleaner production-ready dependency graph.

## Major changes

### Firebase-first runtime

- Migrated remaining live runtime flows away from the old legacy backend client.
- Added shared helpers in `src/services/firebaseProfile.js` for:
  - Firestore profile reads and merges
  - Firebase Auth profile updates
  - Firebase Storage avatar and media uploads
- Exported Firebase Storage support from `src/firebaseClient.js`.

### Profile and avatar sync

- `ProfilePage` now uploads profile photos through Firebase Storage.
- `AccountCustomizePage` now saves display name, username, bio, social links, and avatar state through Firebase-backed profile helpers.
- App user state now restores from both Firebase Auth and Firestore profile data so usernames and avatars survive refreshes more reliably.

### Inventors & Investors migration

- Replaced the remaining networking data layer with Firestore-backed reads and writes.
- Moved media uploads for networking profiles onto Firebase Storage.
- Improved discovery filtering and active conversation refresh behavior.
- Preserved the existing UI contract so the current feature pages continue to work against the new backend path.

### Release hardening

- Removed the active runtime dependency on the old backend client package.
- Deleted the old legacy runtime client file.
- Fixed the ESLint 9 flat-config setup so `npm run lint` works again.
- Removed tracked local Vercel env files from the repo path and ignored `.env.vercel*` going forward.

## Validation

- `npm run lint` passes
- `npm run build` passes
- Local production smoke test verified landing and sign-in page routing on the built app

## Release note

`v0.7.0` marks a new prototype phase focused on a cleaner Firebase-native runtime, tighter release hygiene, and better production readiness for the core user and networking experience.
