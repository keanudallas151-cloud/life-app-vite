# v0.6.7 — Profile avatar upload, PostIt avatars, paper texture, code cleanup

## Profile picture upload
- `shapeUser()` in App.jsx now includes `avatarUrl` from auth `user_metadata.avatar_url` so the field is available app-wide.
- `ProfilePage` avatar circle is now a tappable `<button>` with a camera icon overlay. Tapping opens a native file picker.
- Validates file type (JPEG, PNG, WEBP) and size (max 5 MB) before uploading.
- Uploads to the `profile-avatars` storage bucket under `{userId}/avatar.{ext}`.
- Calls the auth profile update API to persist the URL to user metadata.
- `onAvatarChange` callback in App.jsx updates the live user state so the photo appears immediately without a reload.
- Upload progress and inline error messaging are shown in the profile card.

## PostIt feed avatars
- All three avatar circles in PostItFeed (feed card, detail view, comment) now use a shared `PostAvatar` component.
- `PostAvatar` renders the poster's photo if `authorAvatarUrl` is set, otherwise falls back to the two-letter initials.
- `addPost` and `addComment` in `usePostIt.js` now store `author_avatar_url` (from `user.avatarUrl`) on insert so new posts carry the photo.

## Backend migration (20260419_000002_profile_avatars.sql)
- Creates the `profile-avatars` storage bucket (public read).
- Adds per-user upload / update / delete RLS policies (users can only write to their own `{userId}/` folder).
- Adds `author_avatar_url text NOT NULL DEFAULT ''` column to the `posts` table.
- Adds `author_avatar_url text NOT NULL DEFAULT ''` column to the `comments` table.

## Reader paper texture
- `getParchmentBackground()` in Reader.jsx now layers an SVG `feTurbulence fractalNoise` filter (baked as a data URI) over the parchment gradients.
- Produces genuine rough paper grain — not just CSS gradients — in both light and dark reading mode.
- The SVG is inline as a data URI so there is no network request and no external dependency.

## Code cleanup
- Removed duplicate `@media (max-width: 640px) { input, textarea, select { font-size: 16px } }` block from `index.css`. It was a superset already fully covered by the comprehensive typed-input block added in v0.6.6.
- All other flagged CSS classes left intentionally — removing them would break dynamically constructed classNames in JS (domino risk as noted).

## Compatibility note — iOS native packaging
- This app is JavaScript / React / Next.js — no language rewrite is needed to ship on iOS.
- Use Capacitor: `npm i @capacitor/core @capacitor/ios`, `npx cap init`, `npx cap add ios`, then build in Xcode.
- All safe-area, PWA meta, and WKWebView-compatible CSS added in v0.6.6 is already in place.
